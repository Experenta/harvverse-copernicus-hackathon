SHEYLA - Payloads listos para AI SDK / WhatsApp

Base endpoint:
POST /api/sentinel/agent/scenarios

Headers si llamas desde fuera:
Content-Type: application/json
x-sentinel-agent-key: <SENTINEL_AGENT_API_KEY>

Request ejemplo:
{
  "lotCode": "lotetest",
  "scenario": "ndvi_drop_money",
  "demoOverrides": {
    "previousNdvi": 0.65,
    "currentNdvi": 0.45
  }
}

Scenarios disponibles:
- lot_approved
- eudr_blocked
- water_stress
- fungal_risk
- ndvi_drop_money
- roya_risk
- explain_roya
- flowering_positive

Para enviar WhatsApp:
- Usar response.message.body como texto principal.
- Usar response.whatsapp.variables si el proveedor requiere template variables.
- Usar response.knowledge como grounding para AI SDK.
- Usar response.context para datos del lote, score, YieldPredict y proof.

Endpoints extra:
GET /api/sentinel/agent/context?lotCode=lotetest
GET /api/sentinel/agent/knowledge?signal=S2_ndvi_drop
